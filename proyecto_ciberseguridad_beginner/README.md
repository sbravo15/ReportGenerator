# Proyecto de Ciberseguridad Tecnica para Beginner

Actualizado al 8 de abril de 2026.

## Objetivo
Construir una base tecnica real de ciberseguridad (blue team + fundamentos de app/network security) usando marcos reconocidos y protocolos criticos.

Al finalizar, debes poder:
1. Endurecer un host Linux basico.
2. Entender y validar protocolos de seguridad (TLS, SSH, DNSSEC, IPsec, OAuth2/OIDC, SPF/DKIM/DMARC).
3. Crear runbooks operativos basicos (incidentes y vulnerabilidades).
4. Ejecutar laboratorios con evidencia tecnica (comandos, logs, trazas).

## Roadmap sugerido (8 semanas)
1. Semana 1-2: Networking + Linux + logging base.
2. Semana 3: TLS/HTTPS y SSH hardening.
3. Semana 4: DNS/DNSSEC e IPsec/IKEv2.
4. Semana 5: OAuth2/OIDC y seguridad de APIs.
5. Semana 6: Seguridad de correo (SPF, DKIM, DMARC).
6. Semana 7: Vulnerability management + parches.
7. Semana 8: Simulacion de incidente y postmortem.

## Entregables minimos
1. Checklist de hardening aplicado en al menos 1 host.
2. Evidencia de validacion de TLS y SSH (capturas/comandos/logs).
3. Verificacion de un dominio con SPF/DKIM/DMARC.
4. Runbook de incidente ejecutado en tabletop de 30-60 min.
5. Lista priorizada de riesgos con acciones de mitigacion.

## Reglas de trabajo
1. Todo cambio debe tener evidencia (comando + resultado resumido + fecha).
2. Nunca ejecutar pruebas ofensivas en sistemas sin autorizacion explicita.
3. Priorizar controles basicos primero (inventario, MFA, patching, logging).

## Nivel tecnico
Contenido tecnico puro, pero guiado paso a paso para principiante.
